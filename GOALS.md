**Alex Chen** — Senior PM at Apex AI, working on Model Deployment & Developer Experience.

**My scope:** I own the developer-facing product surface for Apex AI's API platform — from onboarding to eval tooling to rate-limit UX. Success at the team level means developers ship their first production integration in under 7 days and our API adoption grows 2× YoY.

**What energizes me:** Shipping a crisp developer experience that makes a hard technical thing feel simple. Running a tight eval cycle where I can point to a specific failure mode we caught before launch. Bringing structure to ambiguous 0→1 decisions.

**What drains me:** Meetings without a clear decision owner. Writing docs that duplicate things already in code. Being asked to prioritize without being given the constraints.

---

### What I Own

| Area | Description |
|------|-------------|
| API Developer Onboarding | End-to-end activation flow from API key creation to first successful inference call |
| Eval Tooling | Developer-facing eval SDK and dashboard for testing model behavior pre-deployment |
| Rate Limit & Quota UX | Surfacing limits, consumption, and upgrade paths without friction |

**Key metrics I care about:**
- **Time-to-first-call (TTFC):** target ≤ 7 days from signup to first production API call
- **Eval coverage rate:** % of model deployments with at least one eval suite attached before launch gate — target 80% by Q3
- **API activation rate (30-day):** % of new accounts that reach 1,000 API calls within 30 days — current 34%, target 45%

---

### 30-60-90 Day Goals

#### Day 1–30: Listen and Map

- [x] Complete 15 developer interviews (mix: first-time users, power users, churned accounts) — capture verbatim blockers
- [x] Audit the current onboarding flow end-to-end as a new user; log every friction point with a severity rating
- [ ] Map the full stakeholder landscape: eng leads, design, DevRel, safety, policy — understand each team's priorities and blockers
- [ ] Read the last 6 months of support tickets and NPS verbatims; identify the top 5 recurring failure modes

**Success looks like:** I can articulate the top 3 activation blockers with supporting quotes and data, and I've met every key stakeholder at least once. No blind spots about the org.

#### Day 31–60: Define and Decide

- [ ] Ship a revised onboarding spec with 3 prioritized improvements targeting TTFC — reviewed and approved by eng lead and design
- [ ] Draft eval tooling 6-month roadmap; run it through `/prd-readiness` and present to leadership
- [ ] Align with safety team on eval coverage requirement for model deployments (target: written into launch gate policy)
- [ ] Run the first `/go-nogo` for Q3's TTFC improvement initiative

**Success looks like:** A signed-off spec in the eng backlog, a public-facing roadmap, and a clear go/no-go decision on at least one scoped initiative.

#### Day 61–90: Execute and Show Value

- [ ] TTFC improvement v1 shipped — measure pre/post activation rate delta at 30 days
- [ ] Eval coverage rate above 60% (up from ~30% baseline) — driven by tooling improvements and gate policy
- [ ] 3 developer case studies documented in Knowledge/Research/ with provenance tags
- [ ] Present Q3 results to skip-level with a PD-TOL format: Problem → Decision → Tradeoff → Outcome → Learning

**Success looks like:** Activation rate trend is moving (even if not yet at 45%). Eval coverage is measurably up. I have one shipped thing I can point to.

### Personal Development Goals

- Build genuine technical depth in LLM eval methodology — able to write a criteria rubric and grade a suite independently, not just commission one
- Develop a reputation for specificity: every claim I make in a review or strategy doc has a number, a quote, or a source attached
- Improve written communication for a technical audience — less PM-speak, more precision

---

### Strategic Alignment

**OKR ladder-up:** Company OKR: "Become the default API for production AI applications." Team OKR: "Reduce time-to-production for new API developers." My 90-day goal (TTFC ≤ 7 days) directly moves the team OKR's leading metric.

**Single proof metric:** API 30-day activation rate → **45% at day 90** (from 34% baseline). If this number moves by ≥5pp with the onboarding changes live, the quarter succeeded.

**Goal 1 kill condition:** If Priya kills or deprioritizes the TTFC initiative in favor of the enterprise onboarding track, I pivot the 60-day spec work to the enterprise funnel and redefine my proof metric around enterprise time-to-contract.
