# Meetings/

Meeting notes and prep files, one file per meeting: `YYYY-MM-DD_<meeting-slug>.md` (use `Templates/1on1-notes.md` or `Templates/interview-notes.md`).

Read by `/meeting-prep` (upcoming-meeting preparation) and the `meeting-scanner` sub-agent (schedule, action items, interaction history).

Meeting content is private — keep real notes in your private fork or local copy. This README ships with the template so the path the docs reference exists on a fresh clone.
