# Decisions — durable operating decisions

Durable, cross-cutting decisions about **how the OS and the working
relationship run** (e.g. "we always run /peer-review before sharing a PRD",
"backlog grooming happens Fridays").

> For **product / strategy** decisions, use `Knowledge/Decisions/` — that is the
> canonical formal decision layer with provenance tags and reversal conditions.
> This file is only for operating / meta decisions that shape assistant behavior.

## Format

```
### [YYYY-MM-DD] <decision title>
- **Decision:** what was decided
- **Why:** the reasoning
- **Revisit when:** the condition that would reopen it
```
