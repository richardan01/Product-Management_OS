# Quarterly Planning Workflow

This workflow guides the quarterly planning process for creating OKRs and roadmap priorities.

## When to Trigger

Say "quarterly planning" or "Q planning" during the last 2 weeks of each quarter to prepare for the next.

## What This Produces

- Review of previous quarter's OKR performance
- Stakeholder input synthesis
- Draft OKRs for the upcoming quarter
- Aligned roadmap priorities

## Key Constraints

- OKRs should be outcome-based, not output-based
- Limit to 3-4 objectives max
- Key Results must be measurable
- Roadmap should tie directly to OKRs
- Get leadership alignment before finalizing

## Files in This Workflow

1. `workflow-spec.md` - Full process overview
2. `1-review-last-quarter.md` - Step 1: Review previous quarter performance
3. `2-gather-inputs.md` - Step 2: Collect stakeholder input
4. `3-draft-okrs.md` - Step 3: Draft the OKRs
5. `4-review-and-finalize.md` - Step 4: Review and lock with leadership
6. `okr-template.md` - OKR format template

## Timeline

Typical quarterly planning takes 2 weeks:
- Week 1: Review + gather inputs
- Week 2: Draft + review + finalize
